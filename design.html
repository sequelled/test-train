Database Design

We develop normalised databases that are based on ER model. Our motto is to create simple & clean databases, where data is broken into logical pieces without any duplicates and is related to each other with well defined keys and constraints. A well designed database goes a long way in product development, where the code almost feels to write itself.


Along with developing databases with schema, table, columns, we also help to add advanced features such as:

  • Adding rules
  • Creating functions, triggers, trigger functions
  • Creating views
  • Adding foreign keys
  • Adding constraints
  • Adding indexes (single/multicolumn, indexes on expressions, partial indexes)
  • Transactions
  • Using Window functions
  • Inheritance
  • Table partitioning
  • Using foreign data wrappers
  • Using procedural languages (PL/pgSQL, PL/Python, PL/v8)
